Squarespace is a website building and commerce platform that provides REST Commerce APIs to manage merchant site data (products, orders, inventory, transactions) and site/content APIs to read site data. The REST API base URL is
https://api.squarespace.com/{api-version}
and All requests require a Bearer token (API key or OAuth access token)..
dlt is an open-source Python library that handles authentication, pagination, and schema evolution automatically. dlthub provides AI context files that enable code assistants to generate production-ready pipelines. Install with
uv add "dlt[hub]"
and start loading Squarespace data in under 10 minutes.

How do I authenticate with the Squarespace API?
Commerce API requests must include an Authorization header with a Bearer token (API key or OAuth access token). Example: Authorization: Bearer YOUR_SECRET_API_KEY. Include a User-Agent header for identifying your application.
1. Get your credentials
Sign into your Squarespace account and open the Developer tools panel.
Navigate to Developer API Keys.
Click
Generate Key
, give it a name, and select required permissions such as Orders, Inventory, Transactions, etc.
Copy the generated API key (it is shown only once).
Store the key securely and use it as a Bearer token in API requests.

Troubleshooting
Authentication failures
If you receive 401 Unauthorized, verify the Authorization header is present and uses a valid Bearer token. API keys are shown only once at creation—if lost, generate a new key. 401 can also indicate expired or revoked OAuth tokens.
Permission / Forbidden errors
403 Forbidden indicates the token lacks required permissions for the requested Commerce scope (Orders, Inventory, Transactions, etc.). Ensure the API key has appropriate scopes in Developer API Keys settings.
Rate limiting & 5xx server errors
Commerce APIs use standard HTTP codes. For 5xx errors include the
contextId
in support requests. Retry with exponential backoff for transient 5xx errors. Check response body for an error object with
type
,
subtype
,
message
, and
contextId
.
Pagination
Many list endpoints return a top‑level array field (e.g.,
products
,
orders
) plus a
pagination
object with
hasNextPage
,
nextPageCursor
, and a pre‑built
nextPageUrl
. Use the cursor parameter (or
nextPageUrl
) to iterate pages.
Response shape quirks
Squarespace always returns a JSON error object with fields (type, subtype, message, details, contextId). Array fields with no data may be returned as null or an empty array—code should handle both.
Ensure that the API key is valid to avoid 401 Unauthorized errors. Also, verify endpoint paths and parameters to avoid 404 Not Found errors.
